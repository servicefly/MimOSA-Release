## 1. Summary

M6.4 wires M6.1–M6.3 into one capability and exposes it to the user. The
`ResearchEngine` runs **search → budget → synthesize** in a single call with
privacy-aware provider routing; the `ResearchSkill` is the user-facing entry
point; and the intent router + app config integrate it like every other skill.

Key properties:

1. **One call, three stages** — `ResearchEngine.research()` searches, negotiates
   the budget (recording the spend for cost learning), and synthesizes a
   balanced answer. Never raises.
2. **Privacy routing** — a sensitive query is synthesized by a *local* provider
   via the Privacy Guard (M5.4); routing failure fails **safe** (extractive, no
   cloud), never leaking a private query.
3. **Safe-by-default skill** — the default engine is **offline**; the skill
   explains how to enable web search instead of making a surprise network call.
   The host app injects an online engine only when the user opts in.

## 4. Privacy & resource notes

- **Privacy-first default.** `ResearchSettings.web_search_enabled = False`; the
  skill's default engine is offline. No network until the user opts in.
- **Sensitive queries stay local.** The engine routes flagged queries to a
  local provider; routing failure falls back to the local extractive path
  rather than the cloud default.
- **Bounded & honest.** Spend is negotiated and reported; perspective gaps are
  surfaced. Every stage degrades gracefully; `research()` never raises.
