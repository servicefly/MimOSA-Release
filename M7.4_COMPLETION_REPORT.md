## 1. Summary

M7.4 is the conversational front door to everything M7.1–M7.3 built. It adds a
local, LLM-free skill that understands *"what are you working on?"*, *"pause
that"*, *"carry on"*, and *"cancel the download"*, routes those utterances to the
queue, and threads new `TasksSettings` through the app config. With this
milestone the advanced-features stack is usable end-to-end from speech.

Key properties:

1. **Zero-cost, local intent.** A scoped set of regex patterns classifies
   task/queue utterances before the LLM fallback, and deliberately **does not**
   hijack system-control phrases like *"stop the music"*.
2. **Natural disambiguation.** Pause/resume/cancel work for *everything*, a
   *single* task, or *by name* (substring then word-overlap matching, with
   whole-word boundaries so short names don't false-match). Genuinely ambiguous
   requests ask a clarifying question instead of guessing.
3. **Settings-driven.** A `TasksSettings` section (enabled, max-concurrent,
   monitoring, thresholds, error-learning) is validated, clamped, and
   round-trips through `to_dict`/`from_dict`, with old configs loading defaults.

## 2. What was delivered

### Task-control skill (`mimosa/skills/task_skill.py`)

| Capability | Detail |
|------------|--------|
| `TaskControlSkill` | `BaseSkill`, name `"tasks"`, intents `["task_control"]`, `uses_llm = False` |
| Queue | lazy in-memory `queue` property (injectable for host app / tests) |
| Matchers | `_PAUSE_RE`, `_RESUME_RE`, `_CANCEL_RE`, `_STATUS_RE`, `_ALL_RE` |
| Dispatch | `handle()` orders cancel → resume → pause → status (most-specific first) |
| Disambiguation | `_match_task()` — whole-word name match, then >3-char word overlap; ambiguous → `success=False`, `reason="ambiguous"` |
| Generic resume | a bare *"carry on"* with multiple paused tasks resumes **all** ("continue what you were doing") |

### Router wiring (`mimosa/core/intent_router.py`)

- `INTENT_TASK = "task_control"` added to `SUPPORTED_INTENTS`.
- `_TASK_PATTERNS` — scoped to task/queue vocabulary + pause/resume phrasing.
- Classified **before** application/greeting patterns (so *"what tasks are
  running?"* isn't mistaken for *"what apps are running?"*) and before the
  question-shape heuristic.
- `TaskControlSkill()` registered in the default skill list (after research,
  before question).

### Config (`mimosa/utils/config.py`)

- New constants: `DEFAULT_MAX_CONCURRENT_TASKS=2` (min 1 / max 8),
  `DEFAULT_TASK_CPU_THRESHOLD=85.0`, `DEFAULT_TASK_MEM_THRESHOLD=85.0`,
  `MIN_RESOURCE_THRESHOLD=10.0` / `MAX_RESOURCE_THRESHOLD=100.0`.
- `"tasks"` added to `DEFAULT_SKILL_ORDER` (between research and question).
- `TasksSettings` dataclass — `background_tasks_enabled`, `max_concurrent`,
  `resource_monitoring`, `cpu_threshold`, `mem_threshold`, `learn_error_fixes`;
  `validate()` clamps/coerces; wired into `AppConfig` (field + validate +
  `to_dict` + `from_dict`).

## 4. Privacy & resource notes

- **Local & free.** Intent classification and the skill are pure local logic; no
  network, no LLM call for task control.
- **Conservative scope.** Task patterns are intentionally narrow to avoid
  stealing system/media commands.
- **User in control.** Settings let the user disable background tasks, resource
  monitoring, or error-learning entirely; all default to privacy-safe values.
