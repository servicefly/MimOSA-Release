## 1. Summary

M2.1 delivers MimOSA's first **system-integration** capability: managing files and
folders by voice, entirely on the local machine. A new `FileOperationsSkill` performs
**search, open, create, move/rename, and delete** operations, backed by a dedicated
**safety sandbox** (`file_safety.py`) that confines every action to the user's home
directory and refuses to touch system-critical paths.

No file contents ever leave the device ‚Äî file operations are a **Tier-1 (regex) intent**
handled locally with **zero LLM calls**.

## 4. Safety guardrails

- **Home-directory sandbox** ‚Äî operations confined to `$HOME` (plus `/tmp`, `/media/$USER`).
- **System blacklist** ‚Äî `/etc`, `/bin`, `/usr`, etc. are refused outright.
- **Traversal defense** ‚Äî paths fully resolved/normalized before validation; `../` escapes blocked.
- **Sensitive dotfiles** ‚Äî `.ssh`, `.config`, etc. flagged and handled with extra care.
- **Confirmation for destructive actions** ‚Äî delete and overwriting move are two-step (queued `_PendingAction`, "yes"/"no" on the next turn).
- **Trash by default** ‚Äî deletions go to the Trash, not permanent removal, unless explicitly requested.
- **`MIMOSA_FILE_ROOT` override** ‚Äî lets the test suite sandbox into `tmp_path` for hermetic runs.

`validate_path()` returns a `SafetyDecision` (resolved path, `allowed`, `reason`, `message`,
`sensitive`) rather than raising ‚Äî enabling natural-language responses instead of crashes.
