## 1. Summary

M7.3 lets MimOSA get better at recovering from recurring failures **without any
cloud component**. When a fix resolves an error, that pairing is recorded; the
next time a *similar* error appears, MimOSA can suggest the fix that worked
before. It reuses the M5.2 `PreferenceLearner` as its storage/confidence engine,
so it inherits the same privacy-first, evidence-weighted behaviour.

Key properties:

1. **Generalises across noise.** `normalize_error()` strips the volatile parts of
   an error string (hex addresses, hashes, file paths, quoted literals, bare
   numbers; collapses whitespace; lowercases; truncates) so
   `/home/a/x.txt not found` and `/tmp/other.log not found` map to the **same**
   learnable signature.
2. **Confidence-gated suggestions.** Suggestions are only offered above
   `confidence_threshold` (default 0.6), matching the learner's evidence
   saturation so one-off coincidences don't become "advice".
3. **Safe no-op when disabled.** With no learner wired or learning turned off,
   every method degrades to a harmless no-op — never raises, never blocks.

## 2. What was delivered

### Normaliser & learner (`mimosa/tasks/error_learner.py`)

| Capability | Detail |
|------------|--------|
| `ERROR_LEARNING_CATEGORY` | `"error_fix"` — the `PreferenceLearner` category used for storage |
| `normalize_error()` | hex→`<hex>`, long hashes→`<hash>`, paths→`<path>`, quoted→`<str>`, numbers→`<n>`, collapse spaces, lowercase, truncate to 120 chars |
| `FixSuggestion` | dataclass — `fix`, `confidence`, `signature`, `count` |
| `ErrorFixLearner` | `record_fix()`, `suggest_fix()`, `candidates()`, `explain()`, `forget()`; `available` property |

### Behaviour

| Concern | Detail |
|---------|--------|
| Storage | delegates to `PreferenceLearner.observe()` under `error_fix`, keyed by normalised signature |
| Suggestion | `suggest_fix()` returns the highest-confidence fix above threshold, else `None` |
| Inspection | `candidates()` lists all known fixes for a signature with counts; `explain()` returns human-readable rationale |
| Forgetting | `forget()` clears a signature (privacy / correcting bad advice) |
| Disabled | learner `None` or disabled → all calls no-op; `available` is `False` |

## 4. Privacy & resource notes

- **100% on-device.** Error signatures and fixes are stored only in the local
  preference DB; nothing is transmitted.
- **Generalised, not verbatim.** Normalisation deliberately discards paths,
  numbers, and quoted strings, reducing the chance of storing sensitive
  fragments while improving match rate.
- **User-correctable.** `forget()` lets the user (or host app) erase a learned
  pairing at any time.
