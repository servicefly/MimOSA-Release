**Phase:** 5 (Memory & Context)
**Milestone:** M5.1 — Durable conversation history via local SQLite
**Status:** ✅ Complete
**Branch:** `milestone/m5.1` → merged `--no-ff` into `develop`

---

## 1. Summary

M5.1 gives MimOSA a memory that survives a restart. Conversations are written
to a local SQLite database (`conversations.db`) under the user's data directory,
and the existing in-memory `ConversationManager` transparently rehydrates from
that store on launch. Nothing leaves the device, and the feature degrades to the
old purely-in-memory behaviour whenever a store is not supplied.

Key properties:

1. **Local-only storage** — a single SQLite file in an XDG-resolved data dir;
   no network, no telemetry.
2. **Drop-in for `ConversationManager`** — persistence is opt-in through an
   injected `store=`; existing call sites keep working unchanged.
3. **Private-turn aware** — every message carries an `is_private` flag so the
   privacy guard (M5.4) and exports can reason about sensitivity.

| Capability | Detail |
|------------|--------|
| Resolver | `mimosa/memory/paths.py` — `MIMOSA_DATA` → `XDG_DATA_HOME` → `~/.local/share/mimosa` |

## 4. Privacy & resource notes

- Storage is a single local SQLite file; **no data is ever transmitted**.
- Persistence is gated by `PrivacySettings.persist_conversations`; turning it off
  keeps MimOSA in-memory only.
- `is_private` is stored per message so private turns can be filtered, redacted,
  or kept off any future sync.
- When no `store` is injected the manager behaves exactly as before — zero new
  storage surface for headless / minimal deployments.
