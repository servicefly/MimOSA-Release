## 2. Components

### UI configuration (`mimosa/ui/ui_config.py`)
- `UIConfig` dataclass: window size/position, opacity, color theme, animation
  toggle, always-on-top, target FPS, start-hidden. **Pure — no GTK import.**
- `validate()` clamps every field into safe ranges; `from_dict()` ignores
  unknown keys (forward-compatible).
- `load()` / `save()` are **atomic** (tempfile + `os.replace`) and **never
  raise** — a corrupt/missing file silently yields defaults.
- Local-only path: `MIMOSA_UI_CONFIG` env override → else XDG
  `~/.config/mimosa/ui.json`. **No telemetry, no network.**
- Three bundled themes (`aurora`, `ember`, `mono`) via `COLOR_THEMES` /
  `theme_colors()`.

### State bridge (`mimosa/ui/state_bridge.py`)
- `UIState` enum (IDLE / LISTENING / PROCESSING / SPEAKING / DISABLED) and
  `from_voice_state()` mapping (unknown → IDLE, "stopped" → DISABLED).
- `StateBridge` subscribes to the voice loop and marshals every state change
  onto the GTK main thread via injected `glib.idle_add`; falls back to a
  synchronous call when GLib is absent (headless/tests).
- Every callback is exception-guarded — **a UI fault can never propagate into
  the voice thread.** `glib` and the change-callback are injectable.

### Avatar renderer (`mimosa/ui/avatar_renderer.py`)
- **Pure animation math** decoupled from drawing: `tick(dt)` (dt clamped to
  0.25 s) advances breathing phase and a `TRANSITION_SECONDS = 0.35` state
  cross-fade; `set_state()` / `set_audio_level()` drive visuals.
- Per-state visuals: idle breathing scale, listening ring progress, processing
  "thinking" dots, speaking waveform level, eased color blend between themes.
- `draw(cr, w, h)` **lazy-imports Cairo** and is a no-op when Cairo is absent,
  so importing the renderer never requires native libs.
- Module helpers `ease_in_out` / `lerp` / `blend_color` are independently
  tested.

### Avatar window (`mimosa/ui/avatar_window.py`)
- Guarded GTK import → `HAS_GTK`; `AvatarWindow` subclasses
  `Gtk.ApplicationWindow` **only when GTK is present**, else `AvatarWindow =
  None`. Importing the module is always safe.
- Frameless, transparent (CSS), optionally always-on-top; `DrawingArea` with a
  draw-func bound to the renderer; animation via `GLib.timeout_add` at the
  configured FPS (`start_animation` / `stop_animation`).
- Interactions: drag-to-move (persisted), right-click `PopoverMenu`
  (Settings / Hide / Quit), `Escape` to hide. `set_state()` / `set_audio_level()`
  forward to the renderer.

### Window manager (`mimosa/ui/window_manager.py`)
- **Pure** geometry: `MonitorInfo`, `select_monitor`, `clamp_to_monitor`,
  `resolve_position` (corner presets + explicit coords).
- `WindowManager.query_monitors()` takes an injectable Gdk model and returns
  `[]` on any failure; `startup_position()` / `save_position()` persist via
  `UIConfig`; `apply_to_window()` is GTK-guarded.

### Environment detection (`mimosa/ui/environment.py`)
- `has_display()` (DISPLAY / WAYLAND_DISPLAY), `gtk_available()` (guarded GTK4
  probe), `is_gui_available()` (both), `describe_environment()` for diagnostics.

### Application entry (`mimosa/ui/app.py`)
- `MimOSAApplication`: lazily owns the voice loop; `run()` auto-selects
  `run_gui()` when a GUI is available else `run_headless()`.
- In GUI mode the voice loop runs on a **daemon thread**; a `StateBridge` wires
  voice state → `window.set_state` on the main thread.
- `build_arg_parser()` exposes `--no-gui`, `--check`, `-v`; `main()` is the
  console entry. `APP_ID = "ai.mimosa.Assistant"`.

### Voice-loop integration (`mimosa/voice/voice_loop.py`)
- Added `add_state_listener()` / `remove_state_listener()`; `_set_state()` now
  notifies listeners **best-effort** (per-listener try/except → logged, never
  raised). The voice pipeline is unchanged when no UI is attached.

## 4. Privacy & safety notes

- **No telemetry, no network.** Preferences live only in
  `~/.config/mimosa/ui.json` (XDG-respecting, overridable for tests).
- **UI is fully optional.** With no display or no GTK4, MimOSA runs headless and
  **never imports GTK** — the voice pipeline is unaffected.
- **Thread-safe by construction.** The voice loop runs on a worker thread; all
  GTK touches are marshaled to the main thread via `GLib.idle_add`.
- **UI faults are contained.** State-listener and bridge callbacks are
  exception-guarded so a rendering/UI error can never crash the voice loop.
- Config `load`/`save` never raise; corrupt files degrade to defaults.

## 5. Dependencies (optional — UI only)

```bash
# Debian/Ubuntu (Kubuntu 26.04): system GTK4 + introspection + Cairo
sudo apt install python3-gi gir1.2-gtk-4.0 python3-cairo \
                 libgirepository1.0-dev libcairo2-dev pkg-config

# Or via pip (into the project venv):
pip install "PyGObject>=3.42,<3.52" "pycairo>=1.20"
```

- `PyGObject` is pinned `<3.52`: 3.50.0 builds against GLib/girepository 1.74
  (Debian 12); ≥3.56 requires girepository-2.0 / GLib ≥ 2.80.
- Both are **optional** — absent → automatic headless mode.
