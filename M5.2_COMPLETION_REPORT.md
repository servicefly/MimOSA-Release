## 1. Summary

M5.2 lets MimOSA quietly learn how a user likes things done — which editor they
open, which units they prefer, how they like answers phrased — and recall those
patterns with a calibrated confidence. Observations are aggregated locally in a
SQLite table; nothing is uploaded and the learner can be cleared or asked to
"forget" any single fact at any time.

Key properties:

1. **Evidence-weighted confidence** — a prediction's confidence combines how
   *dominant* a value is against how *much* evidence we have, so a single
   observation never masquerades as certainty.
2. **Explainable** — every prediction can be `explain()`-ed back to its counts.
3. **Local + forgettable** — SQLite only, with `forget()` and `clear_all()`.

## 2. What was delivered

### Preference learner

| Capability | Detail |
|------------|--------|
| Store | `PreferenceLearner` — SQLite `learned_preferences`, PK `(category, key, value)` |
| Record | `LearnedPreference` dataclass (counts, confidence, timestamps) |
| Observe | `observe(category, key, value)` — increments evidence, updates last-seen |
| Confidence | dominance × evidence-saturation factor `1 - exp(-count / saturation)` (default saturation = 3) |
| Predict | `predict`, `predict_value`, `get_preferences`, `all_preferences` |
| Introspect | `explain`, `categories`, `keys` |
| Forget | `forget(category, key[, value])`, `clear_all()` |

The saturation curve means roughly **~5 consistent observations** are needed for
a unanimous pattern to cross the default `0.6` confidence threshold — enough to
avoid jumping to conclusions, low enough to feel responsive.

## 4. Privacy & resource notes

- All learning is **local SQLite**; observations never leave the device.
- Gated by `PrivacySettings.learn_preferences` — disable to stop all learning.
- Fully **forgettable**: `forget()` removes a single fact, `clear_all()` wipes
  the table, supporting a clean "MimOSA, forget that" gesture.
- Storage is tiny (aggregate counts only — no raw transcript is retained here).
