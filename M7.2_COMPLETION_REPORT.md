## 1. Summary

M7.2 lets MimOSA *look before it leaps*: it samples CPU / memory / load, keeps a
short rolling history, **predicts** near-term pressure by linear extrapolation,
and exposes a simple gate the task queue (M7.1) can consult before starting new
work. It degrades cleanly on machines (or test runs) where `psutil` is absent.

Key properties:

1. **Optional dependency, graceful.** A `HAS_PSUTIL` flag guards the import.
   With no `psutil`, samples report `available = False` and the gate **fails
   open** (work is allowed) ‚Äî monitoring never becomes a hard blocker.
2. **Predictive, not just reactive.** `predict_pressure()` extrapolates the
   trend across the recent history so the queue can avoid *starting* a job that
   would push the machine over threshold, not merely react after the fact.
3. **Injectable sampler & clock.** Tests feed scripted snapshots ‚Äî fully
   hermetic, no real hardware sampling required.

## 2. What was delivered

### Snapshot & monitor (`mimosa/tasks/resource_monitor.py`)

| Capability | Detail |
|------------|--------|
| `HAS_PSUTIL` | module flag; import guarded so absence is non-fatal |
| `ResourceSnapshot` | dataclass ‚Äî `cpu_percent`, `mem_percent`, `mem_available_mb`, `load1`, `timestamp`, `available`; `summary` text + `to_dict()` |
| `_psutil_sampler` | default sampler reading `psutil.cpu_percent` / `virtual_memory` / `getloadavg` |
| `ResourceMonitor` | `sample()`, `latest()`, `history()`, `is_busy()`, `predict_pressure()`, `can_start_task(predicted=True)`, `gate()` |

### Behaviour

| Concern | Detail |
|---------|--------|
| Thresholds | `DEFAULT_CPU_THRESHOLD = 85.0`, `DEFAULT_MEM_THRESHOLD = 85.0` (configurable via `TasksSettings`) |
| History | rolling window, `DEFAULT_HISTORY = 10` |
| Prediction | linear extrapolation of CPU & memory trend, clamped to `0‚Äì100` |
| `can_start_task` | False only when **known** to exceed threshold (current or predicted); **fails open** (True) when resources are unknown |
| `gate()` | returns a zero-arg callable suitable for `TaskQueue(resource_gate=...)` |

## 4. Privacy & resource notes

- **Local only.** Resource metrics are read and used in-process; nothing is
  stored or transmitted.
- **Lightweight.** Sampling is on-demand and history is capped at 10 entries.
- **Never a hard blocker.** Missing `psutil`, sampler errors, or unknown metrics
  all resolve to "allow", so monitoring can only *defer* work, never deadlock
  the queue.
