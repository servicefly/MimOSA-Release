## 2. Components

### Viseme mapper (`mimosa/ui/viseme_mapper.py`) — pure leaf
- `Viseme` enum: 10 mouth shapes + `SILENCE`, each with an `openness` hint.
- `DEFAULT_PHONEME_TO_VISEME`: eSpeak-ng **IPA** + ASCII fallback symbols.
- `phoneme_to_viseme()`: robust lookup (exact → decoration-stripped → first
  char → ASCII-vowel heuristic → letter→`ALVEOLAR` → neutral default; `None` →
  `SILENCE`). **Never raises.** Diacritics/stress/length marks are stripped via
  Unicode NFD so decorated variants needn't be enumerated.
- `VisemeMapper`: wraps a table; a custom table is merged **over** the default.
- **No GTK/Cairo/audio imports** — safe on headless machines.

### Phoneme extractor (`mimosa/voice/phoneme_extractor.py`) — voice layer
- Owns the shared timeline types: `PhonemeSpan`, `VisemeFrame`,
  `VisemeTimeline` (`viseme_at`, `frame_index_at`, `window_at` →
  `(current, next, blend)` coarticulation; `source` ∈
  `phonemes`/`amplitude`/`empty`).
- `estimate_phoneme_timings()`: distributes a known audio duration across
  phonemes weighted by viseme class (vowels held longest, bilabial stops
  briefest) — local timing without a forced aligner.
- `parse_piper_phoneme_output()`: permissive parser for JSON object/array/JSONL,
  phoneme strings, and timed `{phoneme,start,end}`/`{phoneme,duration}` objects;
  malformed input → `([], [])`.
- `amplitude_to_viseme_timeline()`: short-time RMS envelope (numpy-accelerated,
  **pure-`struct` 16-bit fallback**) mapped to open/closed visemes.
- `PhonemeExtractor.extract()`: orchestrates piper-output → injected phonemizer
  / Piper voice → amplitude → empty. **Never raises.** Mapper + phonemizer are
  injectable for hermetic tests.

### Audio↔viseme sync (`mimosa/ui/audio_sync.py`)
- `AudioVisemeSync`: maps a monotonic clock to a timeline position
  (`position = elapsed + latency_offset`, latency clamped `[-0.5, 1.0]`).
- `start/stop/pause/resume/resync(position)`; `current_viseme()` /
  `current_window()`; `is_finished()`. Clock is **injectable** → deterministic
  tests with no real time. Inactive → silence (mouth eases shut).

### Mouth animator (`mimosa/ui/mouth_animator.py`)
- `MouthShape` (opening/width/roundness/corner-lift/teeth, self-clamping);
  `VISEME_SHAPES` hand-tuned per viseme; `interpolate_shape()` linear blend.
- `MouthAnimator.update(dt)`: exponential smoothing `1 - exp(-speed·dt)` —
  **frame-rate independent, never overshoots**; `dt` clamped ≤ 0.25 s.
- `set_window(cur, next, blend)` gives anticipatory coarticulation; styles
  `natural`/`cartoon`/`minimal`; speed clamped `[1, 40]`.
- `draw()`: **lazy** Cairo bezier mouth (line when shut; filled lens + lip
  outline + optional teeth band when open). No-op without Cairo.

### TTS (`mimosa/voice/tts.py`)
- `synthesize_with_visemes(text, extractor=None)` → `(wav_bytes, timeline)`.
  Audio behaves exactly like `synthesize()`; visemes come from the loaded Piper
  voice's phonemes when possible, else the audio envelope. Holds no shared
  mutable state (safe to call from the voice worker thread). Extractor imported
  **lazily / under `TYPE_CHECKING`** so importing `tts.py` never pulls in the UI
  stack. Extraction **never raises** — audio is always returned.

### Avatar renderer (`mimosa/ui/avatar_renderer.py`)
- `set_viseme_timeline()` / `clear_viseme_timeline()` / `set_lipsync_enabled()`;
  `from_config()`; props `lipsync_active` / `audio_sync` / `mouth`.
- `tick(dt)` samples the sync window → drives the mouth; eases to silence past
  the timeline end; **exception-safe** (a lip-sync glitch never breaks the frame
  loop). Leaving `SPEAKING` stops sync and clears engagement.
- `_draw_speaking()` draws the mouth when lip-sync is engaged, else the original
  M3.1 speaking bar; optional debug overlay.

### UI config (`mimosa/ui/ui_config.py`)
- New fields: `lipsync_enabled` (bool), `viseme_speed` (clamped 1–40),
  `mouth_style` (`natural`/`cartoon`/`minimal`), `lipsync_latency`
  (clamped -0.5–1.0 s), `lipsync_debug` (bool). Validated, clamped, and
  round-tripped through the same local-only JSON file.

## 4. Privacy & safety notes

- **100% on-device.** Phonemes come from the local Piper/eSpeak engine that
  already synthesizes the audio; the fallback analyzes the audio's own energy.
  **No network calls, no third-party phoneme/viseme APIs, no telemetry.**
- **Never crashes the voice loop.** Viseme extraction and every per-frame tick
  are exception-guarded; on any failure the audio is still returned and the UI
  falls back to the M3.1 speaking animation.
- **Degrades gracefully.** No Piper → amplitude visemes. No Cairo/display →
  mouth draw is a no-op; speech is unaffected. Lip-sync can be disabled in
  config.
- **Thread-safe by construction.** TTS returns fresh per-call WAV + timeline;
  the worker thread produces them while the GTK thread animates a previous one.
- **No new import cycles.** Layering is `viseme_mapper` (pure) ←
  `phoneme_extractor` (voice) ← ui modules; `tts.py` imports the extractor
  lazily.

## 5. Dependencies

- **No new required dependencies.** Piper (`piper-tts`) and pycairo remain
  **optional** (already optional from earlier milestones); numpy is used
  opportunistically for the amplitude envelope with a pure-Python fallback.
- `audioop` is deliberately **not** used (deprecated, slated for removal in
  Python 3.13) — 16-bit RMS is computed with `struct`.
