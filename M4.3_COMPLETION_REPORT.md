## 1. Summary

M4.3 rounds out MimOSA's desktop presence with three companion features, all
optional and all degrading cleanly to nothing on a headless machine:

1. **System-tray companion** — a panel status icon + menu (show/hide avatar,
   open chat, mute, settings, quit) that reflects the assistant's live state.
2. **Optional chat window** — type to MimOSA using the *same* brain as the voice
   loop, so typed and spoken turns share one conversation history.
3. **Sprite / expression layers** — an expression model and composable layer
   stack atop the procedural avatar renderer, including a deterministic blink
   animation and sprite-sheet *metadata* (no pixels loaded in core logic).

As with the rest of the UI package, each feature is a pure, headless-testable
controller plus a thin GTK shell.

## 2. What was delivered

### System-tray companion

| Capability | Detail |
|------------|--------|
| Controller | `TrayController` — status label & icon per `UIState`, tooltip, declarative `menu_items()`, `activate()` dispatch |
| Actions | toggle avatar, open chat, toggle mute, settings, quit — via injectable `TrayCallbacks` |
| Robustness | callback exceptions are logged & swallowed; works with zero callbacks wired |
| Shell | `mimosa/ui/tray.py` — GTK + optional AppIndicator; `create_system_tray()` returns `None` headlessly |

### Optional chat window

| Capability | Detail |
|------------|--------|
| Controller | `ChatController` — routes typed input through the shared `IntentRouter`, records turns in the shared `ConversationManager` |
| Resilience | empty input ignored; router errors surfaced as a failed assistant message (never crashes); graceful "not connected" reply when no router |
| History | visible-log cap with trim, `clear()` (keep memory) vs `reset()` (clear both), `to_transcript()` |
| Shell | `mimosa/ui/chat_window.py` — scrolling `Gtk.TextView` + entry; `open_chat_window()` returns `None` headlessly |

### Sprite / expression layers

| Capability | Detail |
|------------|--------|
| Expressions | `Expression` enum + `from_state` / `from_sentiment` / `from_value` mappings |
| Sprite metadata | `SpriteSheet` — grid geometry, named frames, per-index pixel rects; **no image loading** |
| Layers | `ExpressionLayer` / `LayerFrame` + `compose_layers()` (z-order, hidden/transparent culling, sprite-rect resolution) |
| Controller | `ExpressionController` — state→expression mapping, explicit override, deterministic blink via injectable clock, layer emission |

## 4. Privacy & resource notes

- The chat window reuses the existing local conversation memory — no new storage
  or network surface.
- The tray and chat window are **optional**; on machines without a GTK display
  or tray back-end they simply don't appear, and the headless voice path is
  unaffected.
- Expression logic loads **no image assets** — sprite-sheets are described by
  metadata only, so the core stays light and testable.
