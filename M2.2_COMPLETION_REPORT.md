## 2. Components

### Application registry (`mimosa/system/app_registry.py`)
- Parses freedesktop `.desktop` entries from `~/.local/share/applications`,
  `/usr/share/applications`, `/usr/local/share/applications`, Flatpak/Snap
  export dirs, and `$XDG_DATA_DIRS`.
- Strips `Exec` field codes (`%u %f %U %F %i %c %k` …), restores literal `%%`.
- Honours `Type`, `NoDisplay`, `Hidden`; skips malformed/unreadable files
  without crashing (logged at debug).
- Lazy-built, cached catalog; `refresh()` rebuilds it.
- Fuzzy `find()` / `rank()` / `search()` blending exact, substring,
  token-overlap, keyword, and `difflib` ratio scoring.
- `MIMOSA_APP_DIRS` env override for hermetic tests.

### Application skill (`mimosa/skills/application.py`)
- Launch (detached, `start_new_session=True`, startup liveness check),
  list-by-category, is-running (`psutil`), and close.
- **Closing is confirmation-gated** (mirrors M2.1 delete flow); termination is
  `SIGTERM` → `SIGKILL` fallback, or immediate kill on "force"/"kill".
- Spawn / process-lister / terminator are injectable → fully testable.
- Unknown app names produce a "did you mean …?" suggestion.

### System controller (`mimosa/system/system_control.py`)
- Volume: `wpctl` → `pactl` → `amixer` (get/set/change/mute/toggle).
- Brightness: `brightnessctl` → `xbacklight` (get/set/change).
- Wi-Fi: `nmcli` (status/on/off).
- Battery: reads `/sys/class/power_supply/BAT*` directly (no external tool).
- Backend discovery via `shutil.which`; **graceful "tool not installed"**
  messages instead of crashes; every subprocess call is timeout-bounded (5 s).
- Runner / which / sysfs-root injectable → tests run with zero real tools.

### System-control skill (`mimosa/skills/system_control.py`)
- NL parsing for volume/brightness/Wi-Fi/battery, relative & absolute (%).
- **Wi-Fi-off is confirmation-gated** (it disconnects the user); reversible
  volume/brightness changes act immediately.

### Routing (`mimosa/core/intent_router.py`)
- New intents `application` and `system_control`, Tier-1 regex (zero LLM).
- Order: `time → calculator → weather → file_ops → system_control →
  application → greeting`, so "open my notes file" stays a file op and "turn
  the volume up" is not read as launching an app.
- Both skills implement `has_pending_confirmation()` for yes/no follow-ups.

## 4. Privacy & safety notes

- Both skills set `uses_llm = False`; **no text leaves the device** for these
  operations. Routing is local regex, **zero LLM calls**.
- Apps are validated against the registry **before** launch.
- Disruptive actions (close app, Wi-Fi off) require explicit confirmation.
- All external tooling is optional — MimOSA degrades gracefully when absent.
