## 1. Summary

M4.2 smooths the **first five minutes** of MimOSA and keeps users informed about
new releases:

1. A **first-run setup wizard** walks new users through the few decisions that
   matter most — wake word & voice, privacy posture (cloud / local / fully
   offline), and system-integration toggles — then persists them and marks the
   install configured so the wizard never reappears.
2. A **check-for-updates** action (surfaced on the Settings → About page)
   compares the running version against the latest published release using
   robust SemVer comparison, and **never raises** on network failure.

Both features follow the project's split-architecture rule: all behaviour lives
in pure, headless-testable controllers; GTK is a thin optional shell.

## 2. What was delivered

### First-run setup wizard

| Capability | Detail |
|------------|--------|
| Step model | `welcome → voice → privacy → system → finish`, each a declarative `WizardStep` |
| Controller | `SetupWizardController` — works on a *clone* of the live config; `next/back/goto`, progress, validated `get_value/set_value` |
| Commit semantics | `finish()` writes the edited config **and** sets `first_run_complete=True`; `cancel()` discards |
| Re-use | Field specs are shared with the Settings dialog (`settings_logic.FieldSpec`) |
| Gate | `AppConfigManager.is_first_run()` / `mark_first_run_complete()`; `should_run()` classmethod |
| Headless | `open_setup_wizard()` returns `None` (and marks complete) when GTK is absent |

### Check-for-updates

| Capability | Detail |
|------------|--------|
| Version math | `parse_version`, `compare_versions`, `is_newer` — full SemVer incl. prerelease ordering (a final release sorts *after* its prereleases) |
| Checker | `UpdateChecker` with an **injectable** `fetcher(url, timeout)`; `check()` returns an `UpdateInfo` and never raises |
| Default source | GitHub Releases endpoint (the only networked path; isolated in `_default_fetcher`) |
| UI hook | `SettingsController.check_for_updates(checker=None)` for the About page |

### New / changed modules

| File | Change |
|------|--------|
| `mimosa/utils/updates.py` | **New** — version parsing/compare + `UpdateChecker`/`UpdateInfo` |

| `mimosa/ui/setup_wizard_dialog.py` | **New** — thin GTK shell (`HAS_GTK` guarded) |

| `mimosa/ui/app.py` | `_maybe_run_setup_wizard()` wired into headless + GUI activation |
| `tests/conftest.py` | **New** — autouse config-isolation fixture (no writes to real `~/.config`) |

## 4. Privacy notes

- The update check is the only outbound request, is **opt-in** (user clicks the
  button), contacts only the public releases endpoint, and sends no identifying
  data.
- The wizard's privacy step makes **fully-offline (`provider = none`)** a
  first-class, one-click choice.
