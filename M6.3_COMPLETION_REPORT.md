## 1. Summary

Research is token-hungry: every source costs prompt tokens and a thorough answer
costs completion tokens. M6.3 makes that spend **explicit and negotiable**
instead of silent. It estimates the cost, trims to fit a budget while preserving
perspective diversity, and **learns per-topic cost patterns** (building on the
M5.2 preference learner) so repeat research proposes a sensible default budget.

Key properties:

1. **Accurate-or-graceful counting** — `tiktoken` when available, else a
   deterministic `ceil(len/4)` character heuristic. No hard dependency.
2. **Negotiation that preserves balance** — when over budget the negotiator
   first shrinks per-source excerpts, then drops the *least valuable* source,
   preferring to drop over-represented categories so rare perspectives survive.
3. **Cost-pattern learning** — actual spend is bucketed (small/medium/large/
   xlarge) and recorded per topic; `suggest_budget()` proposes a budget from the
   learned pattern. No-ops cleanly when no learner is wired.

| Capability | Detail |
|------------|--------|

## 4. Privacy & resource notes

- **Local & deterministic.** The negotiator makes **no** network or LLM call; it
  only plans the spend. Counting is local.
- **No hard dependency.** `tiktoken` is optional — the heuristic keeps the
  feature working on a bare install (tests exercise both paths).
- **Learning is on-device & optional.** Cost patterns are stored via the same
  local SQLite preference learner as M5.2; with `learn_cost_patterns` off (or no
  learner injected) nothing is recorded.
