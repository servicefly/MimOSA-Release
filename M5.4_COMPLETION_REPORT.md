## 1. Summary

M5.4 makes MimOSA privacy-aware at the moment a query is formed. A hybrid
detector inspects each prompt and, when it looks sensitive (medical, financial,
legal, credentials, personal identifiers…), automatically routes it to a
**local** model so it never reaches a cloud provider. The guard is layered,
explainable, learns user-specific private terms, and can fail safe.

Key properties:

1. **Hybrid tiered detection** — regex (high-confidence PII/secrets) → keyword
   categories → user-learned private terms → optional *local* LLM classifier.
2. **Automatic local routing** — `create_provider_for()` hands back a provider
   already forced local when a query is sensitive.
3. **Explainable & tunable** — every assessment lists why; `fail_safe_private`
   biases toward local on uncertainty.

## 4. Privacy & resource notes

- The guard's entire purpose is to **keep sensitive prompts on-device**:
  detected-private queries are forced to a local provider.
- The optional LLM classifier tier is **local-only** and off by default — the
  guard never calls a cloud model to decide privacy.
- Gated by `PrivacySettings.auto_private_mode`.
- User-learned private terms live in the local preferences DB (M5.2) and are
  fully forgettable.
- `fail_safe_private` lets deployments choose "when unsure, stay local".
