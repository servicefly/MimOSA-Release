## 1. Summary

M6.1 turns a research query into a de-duplicated, **perspective-labelled** list
of sources, ready for budgeting (M6.3) and synthesis (M6.2). It is built to be
honest about *where* information comes from: every source is classified into a
structural category (mainstream / alternative / social / video / think-tank /
academic / official / reference) so the rest of the pipeline can present a
*spread* of viewpoints rather than a single voice.

Key properties:

1. **Pluggable backends** — `StaticBackend` (offline, deterministic) and
   `DuckDuckGoBackend` (key-free HTML endpoint). Backends never raise; any
   failure degrades to an empty list.
2. **Offline-first** — with no backend wired the client returns no sources and
   the engine reports that cleanly. No surprise network calls.
3. **Balance by construction** — automatic domain → category classification
   (curated offline table + TLD heuristics), de-duplication, and an optional
   per-category cap so one perspective can't crowd out the rest.

## 2. What was delivered

### Source model & classification (`mimosa/research/sources.py`)

| Capability | Detail |
|------------|--------|
| Categories | `SourceCategory` enum: mainstream, alternative, social, video, think_tank, academic, official, reference, other |
| Source | `Source` dataclass — auto-classifies category & domain in `__post_init__`; `.text`, `.perspective`, `.to_dict()` |
| Labels | `PERSPECTIVE_LABELS` + `perspective_label()` — human-readable labels per category |
| Classify | `classify_domain()` / `classify_url()` — curated `_DOMAIN_TABLE` + `_SUFFIX_RULES` (`.gov`→official, `.edu`→academic, `.ac.uk`, `.mil`, `.int`) |
| Balance | `summarize_perspectives()` — counts, present, **missing** (gap reporting), labels, diversity, total |

### Search (`mimosa/research/search.py`)

| Capability | Detail |
|------------|--------|
| Raw result | `SearchResult` dataclass + `.to_source()` |
| Contract | `SearchBackend` ABC — `search()`, `available()`; must never raise |
| Offline | `StaticBackend` — list or `by_query` dict; coerces dict/tuple/`Source`/`SearchResult` |
| Online | `DuckDuckGoBackend` — key-free POST to `html.duckduckgo.com`, `bs4` parse + regex fallback, DDG redirect-unwrap, graceful degradation |
| Facade | `SearchClient` — enriches → de-dupes (URL + title/domain) → optional per-category cap → re-ranks; `.online` property |
| Deps | `HAS_REQUESTS`, `HAS_BS4` flags — feature works without either |

## 4. Privacy & resource notes

- **Outbound, opt-in.** Search is the only networked part of M6. The default
  `SearchClient(backend=None)` makes **no** network calls; the host app wires a
  live backend only when the user enables web search.
- **Minimal egress.** Only the query string is sent, with a generic user-agent;
  no identifiers, history, or telemetry.
- **Graceful degradation.** Missing `requests`/`bs4`, network errors, HTTP
  errors, and parse failures all degrade to "no sources", never an exception.
