## 1. Summary

M6.2 produces a single answer from many sources **without flattening them into
one undifferentiated voice**. It groups sources by perspective, attributes
claims to those perspectives, and is explicit about which viewpoints were *not*
available — so the answer never implies false completeness.

Key properties:

1. **Perspective grouping & labeling** — sources are bucketed by category and
   laid out under human-readable headings (Mainstream media, Academic &
   research, Social media & forums, …).
2. **Two synthesis paths** — an **LLM path** (carefully scoped balance-enforcing
   system prompt) and a fully-local, deterministic **extractive fallback**.
   LLM failure or absence silently falls back.
3. **Gap honesty** — missing perspectives are named in the output; `balanced`
   is only true when ≥ 2 distinct perspectives are present.

## 2. What was delivered

### Synthesizer (`mimosa/research/synthesizer.py`)

| Capability | Detail |
|------------|--------|
| Grouping | `group_by_perspective()` — buckets sources by category, preserving first-seen order |
| Model | `PerspectiveGroup`, `Synthesis` (answer, groups, present/missing perspectives, citations, `used_llm`, `balanced`, metadata, `.to_dict()`) |
| LLM path | `SYNTHESIS_SYSTEM_PROMPT` enforces: answer directly · attribute to perspective · note agreement/disagreement · **name missing perspectives** · stay neutral · don't invent facts |
| Prompt body | grouped sources under `## <label>` headings, excerpts truncated to the negotiated per-source token cap (M6.3) |
| Fallback | `_extractive_synthesis()` — deterministic; ranks each source's snippet by query-term overlap, lays them under perspective headings, appends a "perspectives not represented" note |
| Safety | any `LLMError`/exception or empty response → extractive fallback |

## 4. Privacy & resource notes

- **Text-only to the model.** Only the query and source snippets reach the LLM,
  and only when a provider is supplied. The orchestrator (M6.4) routes sensitive
  queries to a *local* provider via the Privacy Guard.
- **Headless-safe.** The extractive fallback needs no model, network, or
  display — research still produces a useful, balanced answer on a bare install.
- **Bounded.** Excerpts are truncated to the per-source token cap before
  prompting, keeping spend predictable.
