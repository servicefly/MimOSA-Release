## 1. Summary

M7.1 gives MimOSA the ability to run long-running work **in the background**
while staying responsive and conversational. A persistent, local SQLite queue
accepts jobs, runs at most a few at a time, and survives restarts. Every job is
cooperatively cancellable and pausable, so the user can say *"pause that"* or
*"carry on"* mid-task.

Key properties:

1. **Local-first & durable.** Tasks live in a SQLite database under the app
   data dir (`tasks.db`); nothing leaves the machine. A crash/restart re-queues
   any task that was mid-flight (`_recover_orphans`).
2. **Bounded concurrency.** At most `DEFAULT_MAX_CONCURRENT` (2) tasks run at
   once, configurable via `TasksSettings.max_concurrent` (1–8).
3. **Cooperative control.** Handlers receive a `TaskControl` object; they call
   `control.checkpoint()` at safe points, which raises `TaskPaused` /
   `TaskCancelled` so work stops cleanly without leaving partial state running.
4. **Resource-aware (optional).** An injectable `resource_gate` callback can
   defer starting new work under load; it **fails open** (never blocks) if the
   gate errors.
5. **Fully injectable for tests.** Clock and resource-gate are injected, and the
   DB can be `:memory:`, so the entire suite is hermetic and deterministic with
   no threads or sleeps.

## 2. What was delivered

### Task model & control (`mimosa/tasks/task_queue.py`)

| Capability | Detail |
|------------|--------|
| `TaskStatus` | `str, Enum`: `QUEUED / RUNNING / PAUSED / COMPLETED / FAILED / CANCELLED` with `is_terminal` / `is_active` properties |
| `Task` | dataclass — id, name, kind, payload, status, priority, progress, message, result, error, timestamps; `to_dict()` / `_from_row()` |
| `TaskControl` | handed to handlers; `set_progress()`, `set_message()`, `checkpoint()` (raises `TaskPaused`/`TaskCancelled` on request) |
| Signals | `TaskPaused`, `TaskCancelled` exceptions drive cooperative stop |
| `TaskQueue` | SQLite-backed manager (see below) |

### Queue operations (`TaskQueue`)

| Group | Methods |
|-------|---------|
| Lifecycle | `register_handler`, `enqueue`, `get`, `list_tasks`, `active_tasks`, `counts`, `running_count` |
| Control | `pause`, `resume`, `cancel`, `pause_all`, `resume_all` |
| Execution | `process_next`, `drain`, `start` / `stop` (background worker thread) |
| Maintenance | `purge`, `clear_all`, `close` |
| Internals | `_claim_next` (respects max-concurrent + global pause + resource gate + registered-handler), `_recover_orphans` (re-queues RUNNING rows on reopen) |

- **Defaults:** `DEFAULT_MAX_CONCURRENT = 2`, `DEFAULT_POLL_INTERVAL = 0.1`.
- **Path helper:** `mimosa/memory/paths.py` gained `TASKS_DB = "tasks.db"` and
  `tasks_db_path()` so the DB location follows the same app-data convention as
  memory and preferences.

## 4. Privacy & resource notes

- **No network, no telemetry.** The queue is a local scheduler only.
- **Bounded by design.** Concurrency cap + optional resource gate keep
  background work from starving the foreground voice loop.
- **Graceful degradation.** A failing resource gate fails open; a crashed run is
  recovered to `QUEUED`; handler exceptions are captured as `FAILED` with the
  error text rather than crashing the worker.
